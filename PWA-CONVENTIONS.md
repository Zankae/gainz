# Shared conventions — Daily Guide & GAINZ

One standard for both apps, so an agent moving between the two project folders
meets the same rules. Where the apps differ, it is on purpose and noted here.

The two apps are **deliberately different in architecture** and must stay that way:

| | Daily Guide | GAINZ |
|---|---|---|
| Shape | one static `index.html`, no framework, no build | React + TypeScript + Vite |
| Store | `localStorage` (one small state object) | IndexedDB via Dexie (thousands of set rows, indexed queries) |
| Runs on | iPad + Raspberry Pi kiosk | Android phone |
| Service worker | hand-written `sw.js` (static shell) | generated by `vite-plugin-pwa` (fingerprinted assets) |

Neither shape is the "better" one — each is what its own brief requires. Do not
port one onto the other. Everything **below** is shared and should match.

## Theme

- Theme is set with `data-theme="dark" | "light"` on `<html>`, never a body class,
  never by swapping a stylesheet.
- Store a separate preference of `dark | light | system`. The DOM still receives only the resolved `data-theme="dark" | "light"`; when the preference is `system`, listen for `prefers-color-scheme` changes and update the resolved DOM value.
- **Structural tokens** (radius, spacing, nav height, fonts) go on `:root`.
  **Colour tokens** go *only* inside the two `html[data-theme=...]` blocks. A colour
  on `:root` still matches `<html>` in light mode and silently leaks the dark palette.
- The palettes themselves differ by design: Daily Guide is warm amber on near-black;
  GAINZ is light-blue accent with green/red gain/loss. Keep them distinct.

## Base CSS (both apps)

- `* { -webkit-tap-highlight-color: transparent; box-sizing: border-box; }`
- `html { background: var(--bg); }` — kills the flash behind overscroll.
- `body { overscroll-behavior: none; }` — no pull-to-refresh inside an installed PWA.
- `@media (prefers-reduced-motion: reduce) { *{animation:none!important;transition:none!important;} }`
- The viewport meta carries `viewport-fit=cover`; the bottom nav uses
  `env(safe-area-inset-bottom)` and any fixed top uses `env(safe-area-inset-top)`.

## No blocking dialogs, ever

`window.confirm`, `window.alert`, and `window.prompt` have inconsistent behavior and presentation across installed PWAs and embedded webviews. Both apps use an accessible in-app dialog component for every confirmation, especially destructive actions.

## No `innerHTML` with data

User text (task titles, notes, exercise names) is never written through `innerHTML`.
Daily Guide builds DOM with a small `el()` helper that only sets `textContent`; React
does this by default. A test asserts `innerHTML` never appears in Daily Guide's output.

## Offline is a hard requirement

- No CDNs, no remote fonts, no analytics, no runtime network calls. The app must work
  with the network off.
- Bundle fonts locally as woff2 and precache them. Fontsource packages are acceptable because Vite emits local files; runtime font CDNs are not. (GAINZ: Barlow Condensed + Inter. Daily Guide: system font stack.)

## Backups — one envelope shape

Both apps export a single JSON file with the same envelope:

```json
{ "app": "<app-id>", "schemaVersion": 1, "exportedAt": 0, "...": "payload" }
```

- `app` is `"daily-guide"` or `"gainz"`. Restore **rejects a file whose `app` differs**.
- Restore **refuses a `schemaVersion` newer** than the running app understands.
- Bump `schemaVersion` only alongside a real migration.
- GAINZ spreads its tables at the top level (`exercises`, `workouts`, …). Daily Guide
  wraps its single state object under `data`. Same envelope, same guards.

## Versioning and updates

- Daily Guide keeps its app and hand-written cache version in step. GAINZ uses a generated Workbox precache whose revisions come from the built file contents; the package version is only the human-facing release version.
- An update is applied on the next visit and the app shows a small, non-blocking
  "new version ready — reload" prompt. It never force-reloads mid-session, and the
  first install never shows the prompt (that is not an update).

## Installability checklist (both)

Manifest with `name`, `short_name`, `start_url`, `scope`, `display: "standalone"`,
`theme_color`, `background_color`, and a 192 + 512 + maskable-512 icon set.
Both apps use **relative** `start_url`/`scope`/`id` (`./`), because both live under
a GitHub Pages subpath (`…github.io/daily-guide/`, `…github.io/gainz/`). Relative
URLs resolve against the manifest's own location, so they also keep working if an
app ever moves to a domain root. GAINZ derives Vite's `base` from the GitHub repository during Actions builds — see its PWA-HANDOFF deployment section. After install the app
opens on the home screen, with its own icon, no address bar, and works offline.

Service workers require `https://` (or `localhost`). On `file://` registration is
skipped and the app still runs, just without offline install.

## Icons

Both icon sets come from a `make_icons.py` (Pillow) in the project folder: a centred
mark on a rounded surface plaque, plus a full-bleed maskable variant with the mark
kept inside the safe circle. Re-run it to regenerate; don't hand-edit the PNGs.
